shuq express pages README

 

How to use:

  1. Setup: 
    1. (Assuming you have our latest files, woking mongodb, node modules, curl, etc. If you are missing something, install it. "npm install" or "npm update" is a good start for the node modules.)
    2. ​Open a terminal window and run your mongodb server -> "sudo mongod" on unix systems.
    3. Open a second terminal window and navigate to Shuq/backend/ShuqServer. Run "node index.js".
    4. Open a third terminal window. You can use this window to execute curl commands as explained below. The other 2 windows should be left running.
  2. Navigation:
    1. ​(Assuming express is running locally on port 3000). 
    2. In a browser of your choice, navigate to "localhost:3000" to view our express splash page and "localhost:3000/README.html" to view this page.
    3. Navigate to "localhost:3000/<collection>" where "<collection>" is replaced with the name of some mongodb collection in order to view that collection laid out in a table by express.
    4. Navigate to "localhost:3000/<collection>/<entity>", where "<collection>" is a collection name and "<entity>" is an _id of an entry in that collection, to view a representation of that entry's Json.
  3. The curl commands:
    1. ​Post a new entry to a collection: curl -H "Content-Type: application/json" -X POST -d <json> http://localhost:3000/<collection>
    2. Replace an entry in a collection: curl -H "Content-Type: application/json" -X PUT -d <json> http://localhost:3000/<collection>/<entity>
    3. Delete an entry from a collection: curl -H "Content-Type: application/json" -X DELETE http://localhost:3000/<collection>/<entity>
  4. Using the curl commands:
    1. ​In each of the commands above, "<collection>" stands for the name of the desired collection. In the case of POST, that collection need not exist yet.
    2. In each of the commands above, "<entity>" stands for the _id of the desired entry in the specified collection.
    3. In each of the commands above, <json> stands either for an explicit json wrapped in single quotes (e.g. '{"title": "test"}') , or the filename of a file which contains a json, preceded by an @ and wrapped in single quotes (e.g. '@testJson').
      1. The column names of the collection page are determined by the first json added to it. If you put jsons of different kinds in the same collection or if you rearrange the order of fields of jsons in the same collection, the table columns may not be correct for these entries.
      2. An _id and created_at field are automatically assigned to the json. You can specify either of these yourself, but it may change the order the json's fields would otherwise show up in, and therefore the tables may have information in the wrong columns. (In the future we seek to make specifying an _id more convenient.)
